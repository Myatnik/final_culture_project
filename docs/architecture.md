# Архитектура

## Слои

 - **app/ui/** — интерфейс (tkinter)

 - **packages/core/** — переиспользуемое ядро

 - **SQLite (hotel.db)** — хранилище данных

## Компоненты

### app/ui/main.py
Точка входа GUI. Отвечает за отображение и пользовательский ввод. Вызывает функции ядра.

### packages/core/database.py
Работа с БД: создание соединения, инициализация схемы и начальных данных.

### packages/core/services.py
Расчёт стоимости, регистрация гостей, управление статусами, назначение задач, выселение.

## Принципы
 Ядро не зависит от GUI

 Приложение обращается к ядру через публичные функции

 Тесты покрывают ядро независимо от приложения

## Контейнерный запуск

Приложение можно запустить в Docker-контейнере через `docker compose`.

- База данных хранится в volume `db_data` и сохраняется между перезапусками
- GUI пробрасывается через X11 socket (`/tmp/.X11-unix`)
- Переменная `DISPLAY` передаётся из хост-системы

Команды:

 `make compose-up` — собрать и запустить контейнер

 `make compose-down` — остановить и удалить контейнер с данными
