# Система управления отелем

## О проекте

Приложение для автоматизации работы стойки регистрации отеля.

Информационная система предоставляет администратору единую базу данных и наглядный интерфейс для отслеживания статусов жилых комнат в реальном времени, быстрой регистрации гостей и распределения задач между обслуживающим персоналом.

## Проблема, которую решает проект

Ручное управление отелем требует отслеживания множества параметров: статусы номеров, данные гостей, задачи персонала, финансовые операции. Проект автоматизирует эти процессы, предоставляя единый интерфейс для всех операций.

## Быстрый старт

### Локальный запуск

```bash
# 1. Установите Tkinter (если не установлен)
sudo apt install python3-tk

# 2. Установите зависимости проекта
make install

# 3. Запустите приложение
make run
```

## Запуск через Docker

```bash
# Разрешите Docker доступ к дисплею (Linux)
xhost +local:docker

# Запустите контейнер
make compose-up

# Остановите контейнер по завершению
make compose-down
```

### Требования для Docker (Linux)

Если команда `xhost` не найдена, установите:

```bash
sudo apt install x11-xserver-utils
```

## Тестирование 

Проект покрыт тестами двух уровней:

### Smoke-тесты

Проверяют основные сценарии работы приложения:
- Подключение к базе данных
- Регистрация гостей
- Выселение гостей
- Назначение задач персоналу

### Unit-тесты

Проверяют бизнес-логику ядра:
- Расчёт стоимости проживания
- Проверка статуса комнат
- Регистрация гостей с валидацией
- Проверка выселения

```bash
# Запустить все тесты
make test

# Сгенерировать отчёт о покрытии
make coverage
```

Отчёт о покрытии будет доступен в папке htmlcov/

## Архитектура

Проект разделён на два основных слоя:

### Переиспользуемое ядро (packages/core/)

Содержит логику, независимую от интерфейса:
- Работа с базой данных
- Расчёт стоимости проживания
- Управление статусами номеров
- Регистрация гостей и выселение
- Назначение задач персоналу

Ядро можно использовать отдельно от GUI, например, в веб-сервисе.

### Интерфейс приложения (app/ui/)

Реализует пользовательский интерфейс на tkinter:

- Отображение статистики
- Форма регистрации гостей
- Управление задачами персонала
- Поиск гостей

Приложение обращается к ядру через публичные функции, не дублируя бизнес-логику.

## Хранение данных

Данные хранятся в SQLite (hotel.db). База данных инициализируется автоматически при первом запуске.

При желании можно настроить базу данных как удобно.

## Контейнеризация

Приложение можно запустить в Docker-контейнере:

```bash

# Остановка с удалением данных

```
